## 6. Detailed User Flows

### 🔄 **Flow 1: Complete Onboarding (5 Minutes)**

```
Step 1: Landing Page (0:00)
  → Hero: "AI for your WhatsApp Business in 5 mins"
  → CTA: "Start Free Trial"
  → Social proof: "Trusted by 1000+ businesses"

Step 2: Signup (0:30)
  → Form: Name, Email, Phone, Business Name, Password
  → Phone OTP verification (Fast2SMS free)
  → Email verification (optional, can skip)
  → Auto-login

Step 3: Welcome Wizard (1:00)
  → "Let's set up your AI in 3 steps"
  → Industry selection (Retail/Restaurant/Service/Other)
  → Business size (1-10, 10-50, 50+)
  → Primary language (Hindi/English/Hinglish)

Step 4: Knowledge Upload (2:00)
  → Option A: Upload PDF/PPT/DOCX
  → Option B: Manual text entry
  → Option C: Use industry template
  → Add FAQs (Q&A pairs)
  → Add products/services
  → Set business hours

Step 5: AI Prompt Generation (3:00)
  → Loading: "AI is analyzing your business..."
  → Show generated system prompt
  → Allow editing
  → Choose AI personality (Friendly/Professional/Casual)

Step 6: Test Playground (3:30)
  → Chat interface to test AI
  → Try sample questions
  → Refine prompt if needed
  → "Looks good? Connect WhatsApp"

Step 7: WhatsApp Connection (4:00)
  → Show QR code
  → Instructions: "Open WhatsApp → Settings → Linked Devices → Link Device"
  → Auto-detect connection
  → Success animation

Step 8: Dashboard (5:00)
  → "🎉 You're live! AI is now handling your WhatsApp"
  → Show stats (will update as messages come)
  → Trial info: "2 days remaining in trial"
  → Tour of dashboard features
```

### 🔄 **Flow 2: Customer Sends Message**

```
Customer: "Aapke yahan red lehenga milega? Price kya hai?"
       ↓
WhatsApp Web Client (whatsapp-web.js) receives
       ↓
Backend Webhook Triggered
       ↓
Check: Is subscription active?
   ├─ NO → Send pre-set message: "Service unavailable"
   └─ YES → Continue
       ↓
Check: Is AI enabled for this conversation?
   ├─ NO → Notify owner only
   └─ YES → Continue
       ↓
Fetch context:
  - Last 10 messages in conversation
  - Customer info (name, tags, history)
  - Business knowledge base
  - System prompt
       ↓
Detect:
  - Language (Hindi/English/Hinglish)
  - Intent (Query/Order/Complaint/Greeting)
  - Sentiment (Positive/Neutral/Negative)
       ↓
Send to Groq API (Llama 3.3 70B):
  Messages: [system_prompt, ...history, user_message]
       ↓
Receive AI response (1-2 sec)
       ↓
Post-processing:
  - Add greetings/sign-off
  - Add product images if mentioned
  - Add UPI payment link if order detected
       ↓
Send via WhatsApp to customer
       ↓
Save to DB:
  - Message in conversations
  - Update usage counter
  - Log for analytics
       ↓
Push to Owner Dashboard (WebSocket):
  - Real-time message in chat view
  - Notification if action needed
```

### 🔄 **Flow 3: UPI Payment for Subscription**

```
Day 2 (Trial ending tomorrow):
  ↓
WhatsApp Alert to Owner:
  "Hi Rajesh! Aapka WhatsBiz AI trial kal khatam hoga.
   Continue karne ke liye plan choose karo: [link]"
  ↓
Owner clicks link → Pricing Page
  ↓
Selects "Pro Plan ₹1499/month"
  ↓
Payment Page Opens:
  
  Method 1: QR Code Scan
    - Show UPI QR code with amount pre-filled
    - UPI ID: 9315515700-2@ibl
    - Amount: ₹1499
    - Note: "WHATSBIZ-{user_id}-{timestamp}"
  
  Method 2: UPI Deep Link (Mobile)
    - Buttons: GPay / PhonePe / Paytm / BHIM
    - Click → Opens UPI app with pre-filled details
  
  Method 3: Manual UPI Transfer
    - Show: "Send ₹1499 to 9315515700-2@ibl"
    - Note: "WHATSBIZ-{user_id}"
    - Submit UTR after payment
  ↓
User completes payment in UPI app
  ↓
Two-step verification:
  
  AUTOMATIC (preferred):
    - Webhook from UPI gateway (if available)
    - Auto-detect transaction
    - Match by UTR + amount + note
  
  MANUAL (fallback):
    - User submits UTR (12-digit transaction ID)
    - Admin verifies in UPI app/SMS
    - Approves in admin panel
    - Auto-activate subscription
  ↓
Subscription activated:
  - End date = today + 30 days
  - Send invoice via WhatsApp + Email
  - Show success on dashboard
  - "Welcome to Pro! Enjoy unlimited messages"
```

### 🔄 **Flow 4: Subscription Expiry**

```
T-24 hours (Day before expiry):
  → WhatsApp: "Renew karne ka time aa gaya hai!"
  → Email reminder
  → Dashboard banner: "Renew now"
  → SMS: "Subscription expires in 24h"

T-1 hour:
  → Final WhatsApp reminder
  → "1 hour mein service band ho jayegi"

T-0 (Expiry time):
  → AI auto-replies stop
  → Customer message receives:
    "Service temporarily unavailable. We'll get back to you soon."
  → Owner WhatsApp:
    "❌ Subscription khatam. Renew karne ke liye: [link]"
  → Dashboard locked except billing
  → 7-day grace period (data retained)

T+7 days (After expiry):
  → Final reminder
  → Data archived after 30 days
```

## 8. Database Schema

### 📊 **Complete Prisma Schema**

```prisma
// schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// ==================== USERS ====================
model User {
  id              String   @id @default(cuid())
  email           String   @unique
  phone           String   @unique
  password        String
  name            String
  businessName    String
  businessType    String?  // retail, restaurant, service, etc.
  role            Role     @default(CLIENT)
  isVerified      Boolean  @default(false)
  isActive        Boolean  @default(true)
  language        String   @default("hi") // hi, en, hinglish
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt

  // Relations
  subscription    Subscription?
  knowledgeBase   KnowledgeBase?
  whatsappSession WhatsAppSession?
  conversations   Conversation[]
  payments        Payment[]
  contacts        Contact[]
  broadcasts      Broadcast[]
  notifications   Notification[]
  usage           Usage[]
  alerts          Alert[]
}

enum Role {
  CLIENT
  ADMIN
  SUPPORT
}

// ==================== SUBSCRIPTION ====================
model Subscription {
  id                String   @id @default(cuid())
  userId            String   @unique
  user              User     @relation(fields: [userId], references: [id])
  
  plan              Plan     @default(TRIAL)
  status            SubStatus @default(TRIAL)
  
  startDate         DateTime @default(now())
  endDate           DateTime
  trialEndDate      DateTime?
  
  messagesLimit     Int      // monthly limit
  whatsappLimit     Int      // number of WhatsApp accounts
  
  autoRenew         Boolean  @default(false)
  cancelledAt       DateTime?
  
  createdAt         DateTime @default(now())
  updatedAt         DateTime @updatedAt
}

enum Plan {
  TRIAL
  STARTER     // 499/mo
  PRO         // 1499/mo
  BUSINESS    // 3999/mo
}

enum SubStatus {
  TRIAL
  ACTIVE
  EXPIRED
  CANCELLED
  GRACE_PERIOD
}

// ==================== KNOWLEDGE BASE ====================
model KnowledgeBase {
  id                String   @id @default(cuid())
  userId            String   @unique
  user              User     @relation(fields: [userId], references: [id])
  
  rawContent        String   @db.Text  // extracted from PDF/PPT
  systemPrompt      String   @db.Text  // AI generated prompt
  customInstructions String? @db.Text
  
  faqs              Json?    // [{q: "", a: ""}]
  products          Json?    // [{name, price, desc, image}]
  services          Json?    // [{name, price, duration}]
  businessHours     Json?    // {mon: "9-9", ...}
  holidays          Json?    // ["2024-12-25", ...]
  
  tone              String   @default("friendly") // friendly, formal, casual
  personality       String   @default("helpful")
  
  uploadedFiles     Json?    // [{name, url, type, size}]
  promptVersion     Int      @default(1)
  promptHistory     Json?    // [{version, prompt, date}]
  
  createdAt         DateTime @default(now())
  updatedAt         DateTime @updatedAt
}

// ==================== WHATSAPP SESSION ====================
model WhatsAppSession {
  id              String   @id @default(cuid())
  userId          String   @unique
  user            User     @relation(fields: [userId], references: [id])
  
  phoneNumber     String?
  sessionData     String?  @db.Text  // encrypted session JSON
  qrCode          String?  @db.Text
  
  status          WAStatus @default(DISCONNECTED)
  lastConnected   DateTime?
  lastDisconnect  DateTime?
  
  isAIEnabled     Boolean  @default(true)
  workingHours    Json?    // schedule
  awayMessage     String?
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}

enum WAStatus {
  DISCONNECTED
  CONNECTING
  QR_READY
  CONNECTED
  AUTHENTICATED
  ERROR
}

// ==================== CONTACTS (CRM) ====================
model Contact {
  id              String   @id @default(cuid())
  userId          String
  user            User     @relation(fields: [userId], references: [id])
  
  phone           String
  name            String?
  email           String?
  tags            String[] // ["lead", "customer", "vip"]
  notes           String?  @db.Text
  customFields    Json?
  
  totalMessages   Int      @default(0)
  lastMessageAt   DateTime?
  firstContactAt  DateTime @default(now())
  
  totalOrders     Int      @default(0)
  totalRevenue    Float    @default(0)
  
  conversations   Conversation[]
  
  @@unique([userId, phone])
  @@index([userId])
}

// ==================== CONVERSATIONS ====================
model Conversation {
  id              String   @id @default(cuid())
  userId          String
  user            User     @relation(fields: [userId], references: [id])
  contactId       String
  contact         Contact  @relation(fields: [contactId], references: [id])
  
  customerPhone   String
  customerName    String?
  
  status          ConvStatus @default(OPEN)
  isAIEnabled     Boolean  @default(true)
  isPinned        Boolean  @default(false)
  isArchived      Boolean  @default(false)
  
  tags            String[]
  notes           String?  @db.Text
  
  unreadCount     Int      @default(0)
  lastMessageAt   DateTime @default(now())
  lastMessage     String?  @db.Text
  
  sentiment       String?  // positive, neutral, negative
  intent          String?  // query, order, complaint
  
  messages        Message[]
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
  
  @@index([userId])
  @@index([customerPhone])
}

enum ConvStatus {
  OPEN
  PENDING
  RESOLVED
  ARCHIVED
}

// ==================== MESSAGES ====================
model Message {
  id              String   @id @default(cuid())
  conversationId  String
  conversation    Conversation @relation(fields: [conversationId], references: [id])
  
  sender          MsgSender
  senderType      String   // customer, ai, owner, system
  
  content         String   @db.Text
  messageType     MsgType  @default(TEXT)
  mediaUrl        String?
  
  isRead          Boolean  @default(false)
  isDelivered     Boolean  @default(false)
  
  aiConfidence    Float?   // 0-1 score
  aiTokensUsed    Int?
  aiModel         String?
  responseTime    Int?     // milliseconds
  
  whatsappMsgId   String?  // wa message id
  
  createdAt       DateTime @default(now())
  
  @@index([conversationId])
  @@index([createdAt])
}

enum MsgSender {
  CUSTOMER
  AI
  OWNER
  SYSTEM
}

enum MsgType {
  TEXT
  IMAGE
  VIDEO
  AUDIO
  DOCUMENT
  LOCATION
  CONTACT
  STICKER
}

// ==================== PAYMENTS (UPI) ====================

  id              String   @id @default(cuid())
```

```javascript
async function processIncomingMessage(userId, customerPhone, message) {
  // 1. Check subscription
  const isActive = await checkSubscription(userId);
  if (!isActive) return sendExpiredMessage(customerPhone);
  
  // 2. Get/Create conversation
  const conversation = await getOrCreateConversation(userId, customerPhone);
  
  // 3. Save incoming message
  await saveMessage(conversation.id, 'CUSTOMER', message);
  
  // 4. Check if AI is enabled for this conversation
  if (!conversation.isAIEnabled) {
    await notifyOwner(userId, conversation, message);
    return;
  }
  
  // 5. Get context
  const context = await getContext(userId, conversation.id);
  
  // 6. Detect intent
  const intent = await detectIntent(message);
  
  // 7. Build messages for AI
  const messages = [
    { role: "system", content: context.systemPrompt },
    ...context.recentMessages.map(m => ({
      role: m.sender === 'CUSTOMER' ? 'user' : 'assistant',
      content: m.content
    })),
    { role: "user", content: message }
  ];
  
  // 8. Call Groq API
  const startTime = Date.now();
  const response = await groq.chat.completions.create({
    model: "llama-3.3-70b-versatile",
    messages,
    temperature: 0.7,
    max_tokens: 500
  });
  const responseTime = Date.now() - startTime;
  
  const aiReply = response.choices[0].message.content;
  
  // 9. Post-process
  const finalReply = await postProcess(aiReply, intent);
  
  // 10. Send via WhatsApp
  await sendWhatsAppMessage(userId, customerPhone, finalReply);
  
  // 11. Save AI message
  await saveMessage(conversation.id, 'AI', finalReply, {
    aiTokensUsed: response.usage.total_tokens,
    aiModel: 'llama-3.3-70b-versatile',
    responseTime
  });
  
  // 12. Update usage
  await updateUsage(userId, response.usage.total_tokens);
  
  // 13. Real-time update to dashboard
  io.to(`user:${userId}`).emit('message:new', {
    conversationId: conversation.id,
    message: finalReply
  });
}
```

```javascript
// services/whatsapp.service.js
import { Client, LocalAuth } from 'whatsapp-web.js';
import qrcode from 'qrcode';

class WhatsAppManager {
  constructor() {
    this.clients = new Map(); // userId -> Client
  }
  
  async initializeClient(userId) {
    if (this.clients.has(userId)) {
      return this.clients.get(userId);
    }
    
    const client = new Client({
      authStrategy: new LocalAuth({
        clientId: userId,
        dataPath: `./sessions/${userId}`
      }),
      puppeteer: {
        headless: true,
        args: [
          '--no-sandbox',
          '--disable-setuid-sandbox',
          '--disable-dev-shm-usage'
        ]
      }
    });
    
    // QR Code event
    client.on('qr', async (qr) => {
      const qrDataUrl = await qrcode.toDataURL(qr);
      await prisma.whatsAppSession.update({
        where: { userId },
        data: {
          qrCode: qrDataUrl,
          status: 'QR_READY'
        }
      });
      
      // Push to frontend via socket
      io.to(`user:${userId}`).emit('whatsapp:qr', { qrCode: qrDataUrl });
    });
    
    // Authenticated
    client.on('authenticated', () => {
      console.log(`WhatsApp authenticated for ${userId}`);
    });
    
    // Ready
    client.on('ready', async () => {
      const phoneNumber = client.info.wid.user;
      await prisma.whatsAppSession.update({
        where: { userId },
        data: {
          phoneNumber,
          status: 'CONNECTED',
          lastConnected: new Date()
        }
      });
      
      io.to(`user:${userId}`).emit('whatsapp:status', {
        status: 'CONNECTED',
        phoneNumber
      });
    });
    
    // Message received
    client.on('message', async (message) => {
      // Filter: ignore status, groups (optional)
      if (message.from.includes('@g.us')) return;
      if (message.from === 'status@broadcast') return;
      
      const customerPhone = message.from.replace('@c.us', '');
      
      // Process with AI
      await processIncomingMessage(userId, customerPhone, message.body);
    });
    
    // Disconnected
    client.on('disconnected', async (reason) => {
      await prisma.whatsAppSession.update({
        where: { userId },
        data: {
          status: 'DISCONNECTED',
          lastDisconnect: new Date()
        }
      });
      
      // Send alert
      await sendAlert(userId, {
        type: 'whatsapp_disconnected',
        message: 'Aapka WhatsApp disconnect ho gaya hai. Reconnect karein.'
      });
      
      this.clients.delete(userId);
    });
    
    await client.initialize();
    this.clients.set(userId, client);
    
    return client;
  }
  
  async sendMessage(userId, to, message, mediaUrl = null) {
    const client = this.clients.get(userId);
    if (!client) throw new Error('Client not initialized');
    
    const chatId = `${to}@c.us`;
    
    if (mediaUrl) {
      const media = await MessageMedia.fromUrl(mediaUrl);
      return await client.sendMessage(chatId, media, { caption: message });
    }
    
    return await client.sendMessage(chatId, message);
  }
  
  async disconnect(userId) {
    const client = this.clients.get(userId);
    if (client) {
      await client.logout();
      await client.destroy();
      this.clients.delete(userId);
    }
  }
  
  // Restore all sessions on server restart
  async restoreSessions() {
    const sessions = await prisma.whatsAppSession.findMany({
      where: { status: 'CONNECTED' }
    });
    
    for (const session of sessions) {
      try {
        await this.initializeClient(session.userId);
      } catch (error) {
        console.error(`Failed to restore session for ${session.userId}`, error);
      }
    }
  }
}

export default new WhatsAppManager();
```
